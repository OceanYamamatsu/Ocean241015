## 🥠 ① なぜ `secret_recipe` が Cookie 欄に入っているの？

### 💡 理由：

Webサイトでは、**Cookie** という仕組みを使って「ブラウザに小さなデータを保存」します。
たとえば：

* ログイン情報
* 言語設定
* ショッピングカートの中身

などを一時的に覚えておくために使われます。

---

今回の問題は「Cookie Monster’s secret recipe（秘密のレシピ）」というテーマなので、
**「Cookie に秘密が隠されている」** という言葉遊び・ヒントになっていました。

つまり、

> サーバーがユーザーに送る Cookie の中に、こっそり `secret_recipe` というキーで秘密（フラグ）を保存していた

というわけです。

あなたが開発者ツールで Cookie を覗いたのは正解です ✅

## 🔢 ② URLエンコードとは？

### 💡 定義：

**URLエンコード（Percent-encoding）** とは、
「URLで使えない文字（空白や記号など）」を、
`%` と 16進数で表現する方法のことです。

---

### 例：

| 元の文字   | URLエンコード後 |
| ------ | --------- |
| 空白 ( ) | `%20`     |
| `=`    | `%3D`     |
| `?`    | `%3F`     |
| `/`    | `%2F`     |

---

つまり、
あなたが見つけた

```
cGljb0NURntjMDBrMWVfbTBuc3Rlcl9sMHZlc19jMDBraWVzXzJDODA0MEVGfQ%3D%3D
```

の `%3D` は、本来の文字で言うと `=` に相当します。

---

## 🧩 ③ 「%3Dと＝にデコードされる」とは？

URLエンコードされた文字列を**元に戻す（デコード）**ことを指します。
「%3D → =」のように変換します。

---

### 手順としてはこうなります：

1. Cookie値：

   ```
   cGljb0NURntjMDBrMWVfbTBuc3Rlcl9sMHZlc19jMDBraWVzXzJDODA0MEVGfQ%3D%3D
   ```
2. URLデコードすると：

   ```
   cGljb0NURntjMDBrMWVfbTBuc3Rlcl9sMHZlc19jMDBraWVzXzJDODA0MEVGfQ==
   ```

   ✅ `%3D%3D` → `==`

---

## 🧠 ④ どのように「Base64エンコード」と断定したのか？

### 💡 理由は以下の通り：

1. **末尾に「=」や「==」がある**

   * Base64エンコードではデータを4文字単位に揃えるため、足りない分を `=` で埋めます。
     → これはBase64文字列の典型的な特徴。

2. **使われている文字の範囲**

   * Base64は、`A–Z`, `a–z`, `0–9`, `+`, `/`, `=` のみを使います。
   * あなたの文字列もこの範囲内に収まっています。

3. **長さが4の倍数**

   * Base64は4文字単位で出力される形式です。
   * 与えられた文字列もそのルールに沿っていました。

4. **CTFでよく使われる手法**

   * 情報を隠すためにURLエンコード＋Base64エンコードを組み合わせるのは定番です。

---

### 🔍 まとめると

| 特徴                         | 該当？ |
| -------------------------- | --- |
| `=` または `==` で終わる          | ✅   |
| 使われている文字が英数字＋`/`, `+`, `=` | ✅   |
| 長さが4の倍数                    | ✅   |
| CTF的文脈（データ隠蔽）              | ✅   |

→ これらの条件から「Base64」と断定できます。
